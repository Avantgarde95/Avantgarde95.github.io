## CDN (Content Delivery Network)

목적

- 물리적으로 멀리 떨어진 곳에 있는 사용자에게도 빠르게 콘텐츠를 제공하기 위함.

구조

- 각 지역에 캐시 서버(PoP, Points of presence)를 분산 배치해, 사용자와 가까운 캐시 서버에서 원본 서버 대신 콘텐츠를 제공.

캐싱 방식

- Static caching: 원본 서버의 콘텐츠를 미리 캐시 서버에 복사.
- Dynamic caching: 미리 캐시 서버에 복사하지 않고, 요청 시점에 원본 서버에서 콘텐츠를 가져와 캐시 서버에 저장. / 각 컨텐츠는 일정 시간 이후 캐시 서버에서 삭제 가능.
